# Shibboleth für Identity Management 

Jupyterhub benötigt eine Service zur Benutzerauthentifizierung. 
In der Regel reicht der Authenticator, der in Jupyterhub enthalten ist, in den meisten Fällen aus. 
Aber an Hochschulen haben die Teilnehmer ihre eigenen Konten. 
Um die Anmeldung der Teilnehmer zu erleichtern, muss Jupyterhub daher Shibboleth, das Benutzerverwaltungssystem der Hochschule, verwenden. 

Shibboleth Service Provider (SP) ist eine Service für Authentifizierung von Benutzern. 
Es muss auf einem Server deployed werden. Die Daten des Benutzers werden jedoch in einer separaten Service - Shibboleth Identity Provider (IdP) gespeichert. 
Shibboleth IdP werden zentral von der IT-Abteilung der Hochschule verwaltet. 
Die Kommunikation zwischen SP und IdP erfolgt über bwIDM (ein Benutzerauthtifikationssystem).

Weil Shibboleth SP und Jupyterhub sind zwei verschiedene Services, die auf einen Server laufen müssen. 
Daher benötigen wir einen Reverse Proxy (Z.B. Apache). 
Er leitet die Requests des Benutzers zunächst an Shibboleth SP und nach der Authentifizierung an Jupyterhub weiter.
